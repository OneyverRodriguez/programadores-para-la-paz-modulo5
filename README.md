# Programadores para la Paz - Módulo 5

Este repositorio contiene los entregables del módulo 5 del programa Programadores para la Paz, centrados en el diseño de una propuesta de solución para el monitoreo comunitario y la atención de afectaciones socioambientales.

## Propósito del proyecto

El objetivo es documentar y estructurar una idea de sistema de veeduría comunitaria que permita:

- registrar reportes de afectaciones desde el territorio;
- visibilizar problemáticas colectivas con evidencia básica;
- automatizar alertas para el equipo de coordinación;
- priorizar funcionalidades de forma progresiva mediante un backlog y una iteración inicial.

## Contenido del repositorio

La carpeta de la semana 10 incluye los siguientes documentos:

- [semana10/automatizacion-semana10.txt](semana10/automatizacion-semana10.txt): diseño de automatización simple con n8n y Telegram para notificar nuevos reportes.
- [semana10/backlog-semana10.txt](semana10/backlog-semana10.txt): historias de usuario organizadas por prioridad.
- [semana10/iteracion-semana10.txt](semana10/iteracion-semana10.txt): definición de la primera iteración y criterios de terminado.
- [semana10/preguntas-semana10.txt](semana10/preguntas-semana10.txt): respuestas a las preguntas de la semana.
- [semana10/reflexion-semana10.txt](semana10/reflexion-semana10.txt): espacio para la reflexión del proceso.

## Enfoque de la semana 10

La semana se concentró en convertir el problema territorial en una propuesta de producto mínima viable, abordando tres aspectos clave:

1. Captura de reportes desde la comunidad.
2. Generación automática de alertas para facilitar la respuesta.
3. Definición de prioridades y criterios para avanzar en el desarrollo.

Este repositorio funciona como un registro del proceso de análisis, planeación y diseño realizado durante la semana.
